# [프로젝트 명세 및 일정] Phase 1 (정보 과잉 - 심층 서버 등반) 개발 지침서

본 문서는 '무료 수면 요금제(Free-Tier Sleep)' 프로젝트의 첫 번째 게임 씬인 **Phase 1: 심층 서버 등반**의 성공적인 12일 단기 개발을 위한 요구사항 명세, WBS 작업 목록, 그리고 디버깅 예방 가이드를 통합한 최종 문서입니다.

---

## 📋 1. 요구사항 명세서 (Requirements Specification)

### A. 플레이어 조작 및 물리 (Player Controller)
* **이동:** `A`, `D` 키(또는 좌우 방향키)를 이용한 유기적인 좌우 이동을 구현합니다.
* **기본 점프:** `Space` 키 입력 시 점프가 발동하며, 바닥(Ground 레이어) 충돌 시 점프 가능 횟수가 초기화됩니다.
* **더블 점프 (2단 점프):** 플레이어의 피지컬을 극대화하기 위해 공중에서 1회 추가 점프를 허용합니다 (최대 연속 2회 점프).
* **코요테 타임 (Coyote Time):** 발판 끝에서 아슬아슬하게 미끄러져 떨어지더라도 0.15초 이내에 점프를 입력하면 공중 점프가 아닌 '지상 점프'로 판정하여 작동시킵니다. (플레이어 하단에서 `Physics2D.Raycast`를 발사하여 체크)
* **가변 점프 (Variable Jump Height):** 스페이스바를 누르고 있는 시간에 따라 높이가 조절됩니다. 상승 중(`velocity.y > 0`)에 점프 키를 떼면 Y축 상승 속도를 즉시 50%로 감속시켜 가볍게 뛸 수 있도록 제어합니다.
* **화면 경계 클램핑:** 플레이어의 X 좌표가 카메라 좌우 시야 밖으로 나가지 않도록 매 `LateUpdate`에서 클램핑합니다. 경계 접촉 시 X 속도를 0으로 만들어 벽에 박혀 떨리는 현상을 방지합니다.

### B. 카메라 및 레벨 생성 (Camera & Level Generation)
* **카메라 수직 추적:** 카메라는 플레이어의 Y축 좌표를 부드럽게 따라 위아래 양방향으로 추적합니다. 빠른 낙하·상승으로 거리가 벌어지면 추적 속도가 거리 비례로 가산되며, 임계 거리 초과 시 즉시 따라잡아 플레이어가 항상 화면 안에 있도록 보장합니다.
* **청크 기반 절차적 스폰:** 카메라가 위로 올라감에 따라, 뷰포트 상단 외곽 영역에 무작위 X 좌표로 발판 및 적 오브젝트가 주기적으로 스폰됩니다.
* **보조 발판 (Side Platform):** 가로형 화면에서 메인 발판과 같은 Y 부근(±0.5)에 일정 확률로 보조 발판을 추가 생성합니다. X 간격을 4유닛 이상 두어 실수로 떨어졌을 때 짧은 낙하로 회복할 수 있도록 합니다.
* **오브젝트 풀링 (Object Pooling):** 12일간의 최적화를 위해 카메라 하단 경계 밖으로 지나쳐 화면에서 사라진 발판, 적, 탄환 오브젝트들은 파괴(`Destroy`)하지 않고, 전용 큐(`Queue`)로 반환하여 재사용합니다.
* **발판 데스폰 유예:** 이단점프 등 일시적 카메라 상승으로 발판이 즉시 회수되는 문제를 방지하기 위해, 카메라 하단에서 일정 거리(약 12유닛) 아래까지는 발판을 유지합니다.
* **목표 고도 설정:** 씬의 명확한 엔딩을 위해 최종 Y축 목표치(예: 고도 1000)가 존재하며, 도달 시 플레이어 조작이 잠기며 클리어 트리거가 작동합니다.

### C. 기믹 및 방해물 (Obstacles & Enemies)
* **시한부 팝업 발판 (Dynamic Pop-ups):** 윈도우 98 에러창 스타일의 발판입니다. 플레이어가 발을 디디는 순간(`OnCollisionEnter2D`) 1.5초 타이머가 작동하며, 붉게 깜빡이는 연출 후 시간이 다 되면 오브젝트 풀로 반환되어 사라집니다. `PlatformEffector2D`로 위쪽에서만 착지 가능, 옆/아래에서는 통과합니다.
* **넉백 바이러스 (Knocker):** 특정 지형을 좌우로 왕복 정찰하는 적입니다. 플레이어와 충돌 시 데미지 대신 충돌 방향의 반대 대각선 아래쪽으로 강한 물리적인 힘(`Rigidbody2D.AddForce` - Impulse)을 가해 낙사를 유도합니다.
* **무게 텍스트 (Heavy Spam):** 하늘에서 수직으로 뚝뚝 떨어지는 텍스트 덩어리입니다. 피격 시 3초간 플레이어의 `Gravity Scale`을 2배로 증가시키는 코루틴을 실행하여 점프력을 절반으로 무겁게 짓누릅니다.
* **혼란 팝업 (Confusion Bug):** 허공에 정지해 있는 홀로그램 형태의 벽입니다. 충돌 시 2초간 플레이어의 좌우 이동 입력값(`moveInput`)에 -1을 곱해 조작을 강제로 반전시킵니다.
* **알림창 탄환 (Notification Projectiles):** 화면 외곽에서 생성되어 플레이어의 현재 위치를 타겟팅해 비행하는 유도탄입니다. 피격 시 캐릭터의 Y 속도를 0으로 만든 뒤 `Vector2.down` 단발 임펄스로 강하게 끌어내립니다.
* **적 자동 스폰 (EnemySpawner):** 플레이어 고도가 임계값(50/80/150/200)에 도달하면 4종이 순차적으로 자동 스폰됩니다. 카메라 바깥에서 미리 생성되어 화면에 들어올 때 이미 움직이는 상태이며, 고도 증가에 따라 스폰 간격이 단축됩니다. 각 적은 독립된 reduction 값을 갖고 Y=1000에서 동시에 minInterval에 도달하는 선형 감소를 보장합니다.

### C-1. 아이템 시스템 (Items)
LevelGenerator가 발판 생성 시 일정 확률(기본 12%)로 발판 위에 아이템을 함께 스폰합니다. 트리거 충돌 시 효과 발동 후 자기 파괴됩니다.
* **하트 (HeartItem):** 체력 1칸 회복. HP가 최대치이면 추첨 단계에서 자동으로 프리즈/슈퍼점프로 대체됩니다.
* **프리즈 (FreezeItem):** 전역 프리즈 매니저를 통해 일정 시간 모든 적/발판 타이머를 정지시키고 화면을 파랗게 페이드 인합니다.
* **슈퍼점프 (SuperJumpItem):** 강한 위쪽 임펄스 + 무적 + 깜빡임을 부여해 단숨에 위로 올라가는 회수 수단입니다.

전역 효과는 `FreezeManager` 싱글톤이 담당하며, 첫 접근 시 GameObject가 자동 생성되어 풀스크린 Canvas + Image 오버레이를 코드로 만들어줍니다. 각 적/발판 스크립트는 `FreezeManager.IsFrozen` 정적 프로퍼티만 체크하여 동작을 조건부로 건너뜁니다.

### D. 치명적 위험 및 시스템 UI
* **데이터 가비지 컬렉터 (The Chaser):** 맵 최하단에서부터 시각적인 붉은색 글리치 노이즈 파도를 일으키며 일정한 속도로 계속 차오르는 위험 지역입니다. 콜라이더에는 3구역 판정이 있어 상단 마진은 무판정, 중간은 데미지 + 적응형 바운스, 하단 1/3은 즉시 Game Over입니다.
* **적응형 바운스 (Adaptive Bounce):** 가비지 컬렉터 중간 구역에 닿으면 플레이어의 낙하 속도를 그대로 위로 반전시켜 튕깁니다. min/max 클램핑으로 너무 약하거나 화면 밖으로 날아가지 않도록 보장하며, 튕긴 직후 짧은 시간 동안은 가변 점프 감속을 적용하지 않아 발판 높이까지 회복이 가능합니다.
* **단계별 속도 가속:** 텐션 유지를 위해 플레이어의 Y 좌표가 300, 600을 돌파할 때마다 가비지 컬렉터 파도의 상승 속도가 각각 1.1배, 1.2배씩 영구적으로 증가합니다.
* **거리 기반 추적 (Catch-up):** 플레이어가 빠르게 위로 도망쳐 파도와 일정 거리 이상 벌어지면 거리 비례로 파도 속도가 가산되어 따라옵니다. 누적 낙하 거리 폭주를 막아 바운스 보정이 일관되게 작동하도록 합니다.
* **진행도 UI:** 화면 상단에 현재 고도와 목표 고도를 계산하여 다운로드 진행률(%) 형태로 보여주는 슬라이더 바를 배치합니다. 킹받는 연출을 위해 99% 구간에 도달하면 목표 문전 앞에서 게이지가 일시적으로 멈추는 버퍼링 연출이 포함됩니다.

## 🚨 3. 구현 중 예상되는 문제점 및 디버깅 예방 방안

### 문제점 1: 절차적 생성 난수로 인해 '점프로 절대 갈 수 없는 맵'이 생성되는 현상
* **상세:** 완전 무작위 난수(`Random.Range`)로 다음 발판의 X, Y 좌표를 스폰하면, 플레이어가 2단 점프를 뛰어도 물리적으로 도달할 수 없는 공백 구역이 만들어져 유저가 억울하게 낙사하는 버그가 발생합니다.
* **방안:** 완전 무작위 생성을 금지하고, **가장 최근에 생성된 발판의 위치(LastPlatformPos)를 기준점 변수로 저장**해야 합니다. 다음 발판 스폰 시 X축은 기준점 좌우 반경 4~5 유닛 이내, Y축은 플레이어 기본 점프 높이 이하인 3~4.5 유닛 이내로만 난수가 발생하도록 타이트한 제약 조건을 부여합니다.

### 문제점 2: 적에게 맞아 날아가는 도중 허공에서 더블 점프가 초기화되는 오류
* **상세:** 넉백 바이러스나 알림창 탄환에 맞아 대각선 밑으로 날아가는 큰 충격을 받을 때, 플레이어 아래로 쏘는 코요테 타임용 Raycast가 의도치 않게 공중에 스폰되어 떨어지던 다른 에너미나 벽면 레이어에 스치면서 순간적으로 바닥에 닿았다고 착각(`isGrounded = true`)하여 공중 점프 카운트가 초기화되는 현상이 일어납니다.
* **방안:** 플레이어가 피격 넉백 상태에 진입하는 순간(`OnKnockback`) 즉시 `isKnocked = true` 상태 bool 변수를 켭니다. 그리고 **피격 직후 최소 0.2~0.3초 동안은 바닥을 감지하는 Raycast 함수 자체를 리턴(`return`)시켜 무시**하도록 예외 처리를 확실하게 걸어두어야 물리 엔진 버그를 막을 수 있습니다.

### 문제점 3: SpringJoint2D 연결 시 캐릭터가 끼어 우주로 사출되는 물리 글리치 현상
* **상세:** 알림창 탄환의 당기는 장력을 유니티 내장 `SpringJoint2D` 컴포넌트로 연결해 구현하면, 뒤로 당겨지는 와중에 플레이어가 딱딱한 발판 모서리나 벽면 콜라이더 사이에 끼었을 때 유니티 물리 엔진(PhysX)의 연산 한계로 인해 캐릭터가 엄청난 가속도를 받아 화면 밖 우주 공간으로 사출되는 치명적인 글리치가 자주 발생합니다.
* **방안:** 해커톤의 짧은 일정을 고려하여 물리 관절 컴포넌트를 붙였다 떼는 위험한 방식은 배제합니다. 피격 시 **단순히 플레이어의 현재 위치에서 수직 아래 방향 `Vector2.down` 벡터를 추출하고, 여기에 매우 강한 순간 힘(`Rigidbody2D.AddForce(Vector2.down * pullForce, ForceMode2D.Impulse)`)을 단발성으로 먹이는 방식**이 물리적 연산도 안전하고 조작감을 깔끔하게 차단하기에 가장 좋습니다.

### 문제점 4: 시한부 팝업 발판 위에서 유저가 제자리 점프 시 코루틴이 중첩되어 터지는 버그
* **상세:** 발판을 밟았을 때 1.5초 뒤 소멸하는 코루틴(`StartCoroutine(DecayRoutine())`)이 작동하는 구조일 때, 유저가 그 발판 위에서 가만히 있지 않고 스페이스바를 연타하며 연속으로 통통 밟으면 `OnCollisionEnter2D`가 여러 번 중복 호출됩니다. 이로 인해 동일한 소멸 코루틴이 수십 개 켜지면서 발판이 제시간에 안 사라지거나, 오브젝트 풀에 이미 반환된 객체를 재참조하는 메모리 크래시가 납니다.
* **방안:** 발판 스크립트 내부에 `private bool isTriggered = false;` 안전 장치 변수를 선언합니다. 최초 충돌 시 이를 즉시 `true`로 셋팅하고, **`isTriggered`가 이미 `true`라면 아래쪽의 코루틴 실행 코드를 무조건 블로킹(`if(isTriggered) return;`)**하여 단 한 번만 소멸 타이머가 가동되도록 원천 차단해야 합니다.

### 문제점 5: PlatformEffector2D 측면 통과 시 모서리에서 막힘
* **상세:** `PlatformEffector2D.SurfaceArc`를 기본값 180으로 두면 발판의 위쪽 반원 전체가 충돌면으로 분류되어, 측면에서 빠르게 접근할 때 위 모서리 부근 충돌 노멀이 surface로 인식되어 통과가 막힙니다.
* **방안:** `SurfaceArc`를 160 정도로 낮춰 측면 모서리(±10도)는 통과되도록 하고, `UseOneWayGrouping`을 켜 단일 그룹으로 처리해 측면 판정을 안정화합니다.

### 문제점 6: 빠른 낙하 시 카메라가 못 따라가 충돌 모션이 화면 밖에서 발생
* **상세:** 단순 `Mathf.Lerp(transform.position.y, targetY, smoothSpeed * deltaTime)` 방식은 거리에 무관한 일정 비율 추적이라, 누적 낙하 속도가 큰 경우 카메라가 한참 뒤처져 가비지 컬렉터 충돌 순간이 화면 밖에서 발생합니다.
* **방안:** 거리 비례 가산(`adaptiveSpeed = smoothSpeed + distance * catchUpFactor`)을 적용하고, 임계 거리(`maxFollowDistance`) 초과 시 즉시 스냅하여 플레이어가 항상 화면 안에 있도록 보장합니다. 임계 거리는 카메라 시야 절반 이하로 설정해야 효과가 있습니다.

### 문제점 7: 적응형 바운스 직후 가변 점프 감속으로 인해 상승 거리가 절반으로 줄어듦
* **상세:** 가비지 컬렉터에 닿아 위로 튕긴 직후 플레이어가 점프 키를 누르지 않은 상태이면, `lowJumpMultiplier`가 즉시 적용되어 상승 속도가 빠르게 깎입니다. 결과적으로 의도한 바운스 높이보다 훨씬 낮은 곳에서 떨어지기 시작해 발판에 닿지 못합니다.
* **방안:** `BounceUp` 계열 호출 시 짧은 보호 시간(`bounceBoostDuration` ≈ 0.3초)을 기록하고, 그 시간 안에는 가변 점프 감속을 적용하지 않도록 `ApplyBetterJump`에서 분기합니다.
