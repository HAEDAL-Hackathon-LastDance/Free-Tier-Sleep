# [기획서 3] 시스템 로직: Phase 2 (방어 - 펜스 디펜스)

## 1. 핵심 메커니즘: 방화벽 드로잉 (Line Drawing Defense)
*   **개요:** 화면 중앙의 코어(Core)로 몰려오는 적들을 마우스로 선을 그어 막아내는 디펜스.
*   **Unity 구현 로직:**
    *   **선 생성:** 마우스 좌클릭 후 드래그 시 마우스 위치(`Camera.main.ScreenToWorldPoint`)를 추적하여 `LineRenderer`의 포인트를 동적으로 추가.
    *   **물리 충돌:** 선이 그려질 때마다 `EdgeCollider2D`의 포인트 배열을 `LineRenderer`의 포인트 배열과 동기화하여 물리적인 벽을 생성.
    *   **리미트:** 그릴 수 있는 선의 총길이(Point 갯수)를 변수 `maxInk`로 제한. 한도 초과 시 더 이상 그려지지 않음. 우클릭으로 전체 화면의 선을 지움(`Clear()`).

## 2. 페널티 시스템: 의도된 렉 (Simulated Input Lag)
*   이 페이즈의 핵심인 **'과잉 연결로 인한 불편함'**을 마우스 조작감으로 표현.
*   **부하 조건:** 
    1. 화면에 존재하는 적의 수 (`GameObject.FindGameObjectsWithTag("Enemy").Length`)
    2. 현재 그려진 선의 총길이
*   **Lag 구현 방식:**
    *   부하가 커질수록 마우스 포인터를 따라가는 시각적 커서(Custom Cursor)의 `Vector3.Lerp` 속도를 낮춤.
    *   실제 마우스 위치와 게임 내 인식되는 드로잉 위치 간의 딜레이를 발생시켜, 늪에서 마우스를 젓는 듯한 무거운 조작감을 줌.

## 3. 적(Nightmare.exe) AI 및 스폰 패턴
*   **목표:** 무조건 화면 중앙의 코어(Transform)를 향해 직선 이동 (`Vector2.MoveTowards`).
*   **기믹:** 선(EdgeCollider2D)에 부딪히면 튕겨 나가거나(`Bounciness`), 데미지를 입고 소멸. 특정 적은 선을 갉아먹음(LineRenderer 포인트 삭제 로직).
*   **웨이브 (과잉):** 시간이 지날수록 스폰 딜레이가 기하급수적으로 짧아지며 화면 전체를 뒤덮음.

## 4. 클리어 조건
*   **시간 버티기:** 게임 내 타이머가 AM 12:00에서 시작해 AM 06:00에 도달하면 클리어 로직 호출 및 엔딩 씬 재생. 코어의 체력이 0이 되면 게임 오버 후 재시작.
