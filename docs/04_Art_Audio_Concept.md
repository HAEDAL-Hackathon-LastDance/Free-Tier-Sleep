# [기획서 4] 디자인 및 아트/오디오 컨셉 상세

### 여기 내용은 예시고 전부 다 우리 게임의 컨셉에 맞게 수정요망.

## 1. 아트 디렉션 (Art Direction)
*   **기본 톤:** 16비트 레트로 픽셀 아트 + 사이버펑크 디스토피아
*   **캐릭터 (P / Agent 892):** 
    *   Phase 1: 잠옷을 입고 눈 밑에 다크서클이 짙은 인간형 스프라이트.
    *   Phase 2: 육신을 잃고 빛나는 단일 픽셀 코어(Core)로 단순화.
*   **UI/UX (메타픽션 디자인):** 
    *   게임 내 메뉴창, 체력바 등은 윈도우 98 스타일의 고전 OS 창으로 디자인.
    *   데미지를 입을 때마다 화면에 블루스크린(BSOD) 파편이 일시적으로 깜빡임.

## 2. 셰이더 및 포스트 프로세싱 (Visual Effects)
*   **Glitch Effect (글리치 셰이더):** 
    *   Unity Shader Graph를 활용해 씬 전환 시 화면이 가로로 찢어지거나 RGB 색상이 분리되는(Chromatic Aberration) 셰이더 적용.
    *   스토리가 반전되는 엔딩에서는 글리치 효과를 극대화하여 프레임이 깨지는 연출.
*   **Bloom (광원 과잉):** Phase 1의 팝업 광고와 Phase 2의 마우스 드로잉 선(방화벽)에는 Post Processing Volume의 Bloom 효과를 강하게 넣어 시각적 과부하를 유도.

## 3. 오디오 연출 (Audio)
*   **SFX (과잉 효과음):** 
    *   메신저 알림, 윈도우 에러음 띠딩- 소리, 키보드 타건음을 노이즈와 섞어 사용.
    *   Phase 2에서 그려진 방화벽에 적이 부딪힐 때마다 치지직거리는 전기 합선 소리 발생.
*   **BGM 흐름:** 
    *   Intro: 불쾌한 모뎀 다이얼업 소리.
    *   Phase 1: BPM 140 이상의 급박하고 날카로운 8비트 칩튠 사운드.
    *   Phase 2: 숨소리조차 들리지 않는 앰비언트 사운드로 시작해, 점차 PC 쿨러가 미친 듯이 도는 백색 소음이 페이드 인 되며 BGM을 덮어버림.

## 4. 1인 개발 파이프라인 최적화 (AI 활용 방안)
*   **에셋 프루핑:** Aider나 Gemini Pro를 활용해 16비트 스프라이트(캐릭터 달리기, 점프 등)의 베이스 이미지를 프롬프팅으로 빠르게 생성 후, Aseprite나 포토샵에서 리터칭하여 사용.
*   **스크립트 자동화:** 물리 연산(SpringJoint2D) 파라미터 미세 조정 및 LineRenderer 최적화 로직은 ChatGPT Codex, Aider 등의 AI 에이전트를 교차 활용하여 베이스 코드를 작성하고 시간 단축. (예: "유니티 LineRenderer를 이용해 마우스 속도에 비례해 선의 두께가 달라지는 드로잉 스크립트 작성해줘" 등).
