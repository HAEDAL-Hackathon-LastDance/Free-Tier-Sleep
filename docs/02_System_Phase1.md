# [기획서 2] 시스템 로직: Phase 1 (정보 과잉 - 심층 서버 등반)

**개요:** 강제 스크롤되는 무의식의 심층 서버에서, 끝없이 쏟아지는 정보(팝업창)를 밟고 위로 도주하는 **수직 등반형 2D 플랫포머 액션** 씬입니다. 드로잉과 대시 시스템을 배제하고 오직 **점프 피지컬** 하나로 극한의 생존을 요구합니다.

**핵심 목표:** 밑에서 차오르는 데이터 가비지 컬렉터(즉사 판정)를 피해 최대한 높이, 오래 살아남아 상층부로 탈출하는 것.

---

## 1. 플레이어 조작 및 점프 시스템 (Core Mechanics)
가장 순수한 형태의 플랫포머 조작감을 구현합니다. 불쾌한 조작감을 없애고 찰진 손맛을 주입하기 위해 세밀한 물리 보정 로직을 적용합니다.

### A. 코요테 타임 (Coyote Time)
* **규칙:** 발판에서 미끄러져 떨어져 허공에 뜬 상태라도, 아주 짧은 시간(약 0.15초) 동안은 점프 입력을 허용합니다.
* **구현 방식 (Raycast 활용):**
    * 플레이어 캐릭터의 하단(발끝)에서 아래쪽으로 짧은 길이의 `Physics2D.Raycast`를 매 프레임 발사합니다.
    * Ray가 'Ground(발판)' 레이어에 닿고 있으면 `isGrounded = true` 및 타이머 `coyoteTimeCounter = 0.15f`로 갱신.
    * 발판을 벗어나 Ray가 닿지 않게 되면 `coyoteTimeCounter`가 `Time.deltaTime`만큼 감소합니다.
    * 점프 입력 시 `coyoteTimeCounter > 0f`라면 점프를 실행하고 카운터를 0으로 초기화합니다.

### B. 가변 점프 (Variable Jump Height)
* **규칙:** 점프 키(Spacebar)를 짧게 누르면 낮게 점프하고, 꾹 누르면 최대 높이까지 점프합니다.
* **구현 방식:**
    * 플레이어가 상승 중일 때 (`Rigidbody2D.velocity.y > 0`) 점프 키에서 손을 떼면(`Input.GetButtonUp`), 현재 Y축 속도에 0.5f(절반)를 곱해 상승력을 강제로 깎아내립니다.

### C. 더블 점프 (Double Jump)
* **규칙:** 공중에서 점프 키를 한 번 더 누르면 추가 점프가 발동합니다. 최대 2단 점프까지 가능합니다.
* **구현 방식:**
    * `int jumpCount` 변수를 선언합니다. 최대 허용치는 2입니다.
    * Raycast가 Ground에 닿아 `isGrounded == true`가 되면 `jumpCount = 2`로 초기화됩니다.
    * 점프 입력 시 `jumpCount > 0`일 때만 점프가 실행되며, 실행 직후 `jumpCount--` 처리합니다.

```csharp
// 플레이어 점프 구현 의사코드 (Pseudo-code)
if (isGrounded) {
    coyoteTimeCounter = coyoteTime;
    jumpCount = maxJumps; // 더블 점프 횟수 초기화
} else {
    coyoteTimeCounter -= Time.deltaTime;
}

if (Input.GetButtonDown("Jump") && (coyoteTimeCounter > 0f || jumpCount > 0)) {
    rb.velocity = new Vector2(rb.velocity.x, jumpForce);
    jumpCount--;
    coyoteTimeCounter = 0f; // 코요테 타임 소진
}

if (Input.GetButtonUp("Jump") && rb.velocity.y > 0f) {
    rb.velocity = new Vector2(rb.velocity.x, rb.velocity.y * 0.5f); // 가변 점프
}
```

## 2. 레벨 디자인 (Map & Platforms)
12일의 해커톤 일정을 고려하여, 일일이 맵을 찍는 수동 레벨 디자인 대신 **청크(Chunk) 기반 수직 절차적 생성(Procedural Generation)** 맵을 채택합니다.

### A. 맵 생성 규칙 (Procedural Generation)
* **무한 맵 여부:** 완전한 무한 맵이 아니라, **목표 고도(예: Y 좌표 1000)**가 존재하는 유한형 절차 생성 맵입니다. 목표 고도 도달 시 클리어 컷신으로 넘어갑니다.
* **카메라 이동:** 카메라는 플레이어의 Y축을 따라 부드럽게 위로 이동하며, 아래로는 절대 내려가지 않습니다.
* **오브젝트 풀링 (Object Pooling):** 끝없이 생성되는 발판으로 인한 메모리 누수를 막기 위해, 카메라 밖(아래쪽)으로 지나친 발판은 즉시 비활성화 후 재사용 큐(Queue)로 반환합니다.

### B. 발판 규칙: 다이내믹 팝업창 (Dynamic Pop-ups)
* **스폰 방식:** 카메라의 뷰포트 바로 위쪽(플레이어가 곧 도달할 영역)에 무작위 위치로 팝업창 발판이 주기적으로 스폰됩니다. 플레이어의 고도가 높아질수록 발판 간의 거리가 멀어지거나 스폰 간격이 불규칙해져 난이도가 상승합니다.
* **시한부 기믹:** * 팝업 발판 위에는 `[X]` 닫기 버튼이 그려져 있습니다.
    * 플레이어의 발이 닿는 순간(`OnCollisionEnter2D`), 1.5초 타이머가 작동합니다.
    * 타이머 진행 중 발판이 붉게 깜빡이며, 0초가 되면 발판 객체가 파괴(또는 풀링 반환)되어 플레이어가 떨어지게 됩니다. 
    * 빠른 상황 판단과 더블 점프를 이용한 신속한 이탈이 강제됩니다.

## 3. 적(Enemy) 및 방해 기믹 (Obstacles)
대시(무적 회피기)가 없으므로, 피할 수 없는 유도탄 패턴을 배제하고 철저히 **플랫포밍과 점프로 회피하거나 돌파**해야 하는 방해물들로 구성합니다.

* **넉백 바이러스 (Knocker):** * 좌우로 단순 정찰 이동. 부딪히면 데미지 대신 플레이어를 대각선 아래로 튕겨냅니다(Knockback). 
    * `OnCollisionEnter2D` 시, 접촉 방향의 반대 벡터로 `Rigidbody2D.AddForce`(Impulse)를 적용하여 낙사 위험을 극대화합니다.
* **무게 텍스트 (Heavy Spam):** * 하늘에서 수직으로 뚝뚝 떨어지는 스팸 덩어리. 
    * 맞으면 충돌 트리거가 발생하여 3초간 플레이어의 `Gravity Scale`을 2배로 증가시키는 Coroutine이 실행됩니다. 점프력이 절반으로 감소합니다.
* **혼란 팝업 (Confusion Bug):** * 특정 고도에 정지해 있는 홀로그램 적. 
    * 충돌 시 플레이어 스크립트의 `moveInput` 값을 `-moveInput`으로 곱하는 상태 이상 bool을 활성화하여 2초 동안 좌우 이동 키(A, D) 조작을 반전시킵니다.
* **알림창 탄환 (Notification Projectiles):**
    * 화면 밖에서 날아오는 카톡, 이메일 알림창들.
    * 피격 시 캐릭터를 뒤(시작점 방향)로 강하게 넉백시킴 (넉백 벡터에 `SpringJoint2D`의 당기는 힘이 더해져 치명타가 됨).

## 4. 치명적 위험: 데이터 가비지 컬렉터 (The Chaser)
* **역할:** 플레이어가 한 곳에 머물지 못하게 강제하는 타임 어택 요소이자 가장 큰 공포 요소.
* **규칙:** 화면 하단에서부터 붉은색 글리치(노이즈) 파도가 일정한 속도로 차오릅니다. 여기에 닿는 순간 즉시 **Game Over (시스템 삭제)** 처리됩니다.
* **난이도 조절:** * 기본 속도: 플레이어의 걷기 속도보다 약간 느림.
    * 가속도: 플레이어의 Y좌표가 300, 600을 넘길 때마다 파도의 차오르는 속도가 1.1배, 1.2배씩 미세하게 증가하여 후반부 텐션을 올립니다.

## 5. 진행도 및 실패 조건
* **진행도 UI:** 화면 상단에 '다운로드 진행률' 바를 배치. 문(목표 고도)에 가까워질수록 99%에서 멈춰있는 등 킹받는 UI 연출.
* **실패:** 뒤로 튕겨 날아가 가비지 컬렉터(글리치 파도)에 닿거나 화면 좌측/하단 경계(Dead Zone) 밖으로 떨어지면 처음부터 다시 시작.
