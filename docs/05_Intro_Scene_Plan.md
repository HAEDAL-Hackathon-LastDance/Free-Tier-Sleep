# [기획서 5] 인트로 씬 (Intro Scene) 상세 설계서

## 1. 씬 개요
*   **씬 명칭:** `Scene_Intro`
*   **목적:** 게임의 세계관(무료 수면 요금제)을 직관적으로 전달하고, 플레이어를 자연스럽게 Phase 1(정보 과잉 구역)으로 몰입시킴.
*   **핵심 키워드:** 어둠, 과잉 연결(Hyper-connectivity), 강제 광고, 노이즈, 탈출.

## 2. 연출 및 구성 요소

### A. 비주얼 레이아웃
1.  **배경 (Background):** 
    *   좁고 어두운 단칸방. 최소한의 가구(침대, 낡은 컴퓨터 본체)만 실루엣으로 보임.
    *   중앙에는 침대에 누워 있는 주인공 'P'의 스프라이트 배치.
2.  **광케이블 연출:** 
    *   주인공의 머리 뒷부분에서 화면 밖(혹은 천장)으로 이어지는 굵고 투박한 광케이블(랜선) 줄기.
    *   케이블을 따라 미세하게 흐르는 발광 데이터 패킷 효과(LineRenderer 사용 권장).
3.  **팝업 광고 오버레이:**
    *   화면 곳곳에 무작위로 생성되었다가 사라지는 싸구려 네온 컬러의 팝업 광고창들.
    *   내용: "지금 당장 클릭!", "72% 할인!", "당신의 뇌는 안전합니까?" 등 자극적 문구.
    *   연출: 지직거리는 노이즈와 함께 깜빡이며 플레이어의 시야를 의도적으로 방해.

### B. 텍스트 연출 (Typing Effect)
*   **UI 위치:** 화면 하단 중앙, 혹은 주인공 머리 위 (말풍선 형태 제외, 시스템 메시지 느낌).
*   **폰트:** 둥근모꼴 (DungGeunMo).
*   **출력 단계:**
    1.  `[시스템 메시지] 요금제 갱신 실패. 무료 광고 모드로 전환합니다.` (시스템 사운드: 비프음)
    2.  (잠시 정적 후 광고 팝업 폭발적 증가)
    3.  `P: "뇌를 갉아먹는 이 과잉 연결에서... 벗어나야 한다."`
    4.  `P: "저 경계 너머로."`

### C. 사운드 설계
*   **앰비언스:** 낮은 기계음(Humm)과 전자기적 노이즈.
*   **효과음:** 
    *   글자 출력 시 타자기 소리 혹은 디지털 노이즈.
    *   팝업창 등장 시 불쾌한 알림음 (띵!, 띠링!).
*   **전환 사운드:** Phase 1 진입 직전, 귀를 찌르는 듯한 고주파 글리치 사운드.

## 3. 기술적 구현 사양

### A. Glitch Transition (Phase 1 진입)
*   **작동 방식:** 모든 텍스트 출력이 종료된 후 약 1~2초간 화면 전체에 강한 글리치 셰이더 적용.
*   **구현:** 
    *   Post Processing의 `Chromatic Aberration`, `Lens Distortion` 값을 순간적으로 최대치로 조절.
    *   Sprite가 조각나거나 색상이 뒤틀리는 연출 후 검은색으로 페이드 아웃(혹은 화이트 노이즈로 전환).
    *   다음 씬(`Scene_Phase1`) 로드.

### B. 광고 팝업 매니저 (AdPopupManager)
*   **기능:** 
    *   미리 정의된 팝업 이미지/텍스트 중 무작위 선택.
    *   화면 내 랜덤한 좌표에 인스턴스화.
    *   일정 시간 후 파괴 혹은 노이즈와 함께 사라짐.

## 4. 작업 체크리스트

- [ ] 랜선 오브젝트 및 데이터 흐름 파티클/라인렌더러 설정
- [ ] 팝업 광고용 프리팹 (3~5종) 제작
- [ ] 타이핑 효과 스크립트 작성 (TextMesh Pro 활용)
- [ ] 화면 전환용 글리치 셰이더/포스트 프로세싱 설정
- [ ] 씬 로드 및 전환 로직 구현
