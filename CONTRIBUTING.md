# 🤝 프로젝트 기여 가이드 (Contributing Guide)

본 문서는 프로젝트의 효율적인 협업을 위한 표준 개발 전략을 정의합니다.

## 🌿 1. 브랜치 전략
- `main` : 최종 배포용 브랜치 (건드리지 않음)
- `dev` : **기본 브랜치**입니다. 모든 작업의 베이스이자 통합 구역입니다.
- **개발 브랜치** : `타입/이슈번호-설명` 형식으로 생성합니다.
  - 예시: `feat/#12-player-move` (12번 이슈 플레이어 이동 기능 개발)

## 📝 2. 커밋 및 이슈 컨벤션
콜론(`:`) 앞뒤 공백을 반드시 지켜주세요.
- `feat : ` 새로운 기능 개발
- `bug : ` 버그 수정
- `docs : ` 기획서, 리드미 등 문서 작업
- `chore : ` 빌드 설정, 유니티 패키지 추가, 에셋 임포트

## 🚀 3. 전체 작업 프로세스
1. **이슈 생성:** 작업 전 반드시 [Issues] 탭에서 이슈를 먼저 생성합니다.
2. **로컬 최신화:** 작업 전 `git pull origin dev`를 통해 최신 상태를 유지합니다.
3. **브랜치 생성:** 이슈 번호를 따서 브랜치를 만듭니다.
4. **작업 및 푸시:** 유니티에서 작업 후 커밋하고 푸시합니다.
5. **PR 생성:** 깃허브에서 `dev` 브랜치를 향해 PR을 올리고 리뷰를 받습니다.
